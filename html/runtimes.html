런타임

에이전트의 실제 작업을 실행하는 독립 환경

런타임이란? 에이전트 코어는 어떤 작업도 직접 실행하지 않습니다. 모든 실제 작업(파일 읽기, 셸 실행, HTTP 요청 등)은 런타임에 위임됩니다. 런타임은 독립적으로 교체·비활성화·테스트할 수 있습니다.

내장 런타임 목록

ID클래스기본 활성화주요 작업
reasoningReasoningRuntime활성CoT 추론, 반성(reflection)
toolsToolRuntime활성샌드박스 셸 실행
filesFileRuntime활성파일 읽기/쓰기/목록/삭제
memoryMemoryRuntime활성벡터 메모리 저장·검색
apiApiRuntime활성외부 HTTP 요청
browserBrowserRuntime비활성Playwright 브라우저 자동화

런타임별 상세 설명

reasoning — 추론 런타임

Gemma 4에게 체인-오브-씽킹(CoT) 프롬프트를 보내고 추론 결과를 반환합니다. reflect 작업으로 이전 응답에 대한 자기 비평도 지원합니다.

# runtimes.yaml
reasoning:
  enabled: true
  config:
    strategy: chain_of_thought   # chain_of_thought | reflect
    max_reflection_steps: 3

지원 작업: chain_of_thought, reflect

files — 파일 런타임

allowed_roots 밖의 경로는 절대 접근할 수 없습니다. FileAccessError로 차단되고 RuntimeResult(success=False)를 반환합니다.

files:
  enabled: true
  config:
    allowed_roots:
      - ./workspace           # 허용 루트 디렉토리
      - /tmp/agent_workspace
    max_file_size_mb: 50

지원 작업: file_read, file_write, file_list, file_delete

# 런타임 직접 호출 예시 (Python)
from core.runtime.base import RuntimeCall
result = files_runtime.execute(RuntimeCall(
    runtime_id="files",
    operation="file_read",
    params={"path": "./workspace/report.txt"}
))
print(result.data["content"])

api — HTTP 런타임

외부 REST API 호출을 처리합니다. allowed_domains가 비어있으면 모든 도메인 허용, 목록이 있으면 해당 도메인만 허용합니다.

api:
  enabled: true
  config:
    timeout_seconds: 15
    max_redirects: 3
    allowed_domains:          # 빈 리스트 = 전부 허용
      - api.github.com        # 특정 도메인만 허용
      - api.openai.com

지원 작업: http_get, http_post, http_request

memory — 메모리 런타임

기본은 인메모리 저장소입니다. 영구 저장이 필요하면 backend: chroma로 전환하세요.

memory:
  enabled: true
  config:
    backend: chroma            # in_memory | chroma
    persist_path: ./data/memory
    embedding_model: nomic-embed-text

지원 작업: memory_read, memory_write, memory_forget

메모리 범위(scope): session (현재 세션) | agent (에이전트별) | global (전체 공유)

tools — 셸 실행 런타임

샌드박스 모드(sandbox: true)에서는 ;, &&, | 등 위험한 토큰이 포함된 명령을 차단합니다.

tools:
  enabled: true
  config:
    timeout_seconds: 30
    sandbox: true              # false로 변경 시 모든 셸 명령 허용 (주의)

browser — 브라우저 런타임 (opt-in)

Playwright가 설치된 경우에만 동작합니다. 기본 비활성화입니다.

# 활성화 방법
# 1. browser: enabled: true  (runtimes.yaml)
# 2. pip install playwright
# 3. playwright install chromium

지원 작업: navigate, extract_text, screenshot

새 런타임 추가

# 1. runtimes/myruntime/runtime.py 생성
from core.runtime.base import HealthStatus, RuntimeCall, RuntimeResult

class MyRuntime:
    runtime_id = "myruntime"
    capabilities = ["my_op"]

    def configure(self, config: dict) -> None:
        self._value = config.get("key", "default")

    def execute(self, call: RuntimeCall) -> RuntimeResult:
        if call.operation == "my_op":
            return RuntimeResult(success=True, data={"result": "..."})
        return RuntimeResult(success=False, error=f"Unknown: {call.operation}")

    def health_check(self) -> HealthStatus:
        return HealthStatus.OK
# 2. config/runtimes.yaml에 추가
runtimes:
  myruntime:
    enabled: true
    module: runtimes.myruntime.MyRuntime
    config:
      key: custom_value
# 3. 에이전트 YAML에서 접근 허용
runtime_access:
  - myruntime
