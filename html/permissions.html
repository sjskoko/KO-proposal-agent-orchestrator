권한 시스템

능력(Capability)별로 에이전트 접근을 제어합니다

능력(Capability) 목록

Capability설명기본 허용
file_readallowed_roots 내 파일 읽기아니오
file_writeallowed_roots 내 파일 쓰기아니오
shell_exec셸 명령 실행 (샌드박스)아니오
network_access외부 네트워크 접근아니오
mcp_callMCP 서버 도구 호출아니오
api_call외부 REST API 호출아니오
memory_read메모리 스토어 읽기예
memory_write메모리 스토어 쓰기예
sub_agent_delegate서브에이전트에게 작업 위임아니오

내장 권한 세트

세트명file_readfile_writeshell_execnetworkmcpapidelegate
unrestricted✅✅✅✅✅✅✅
worker✅✅❌❌✅❌✅

network_worker✅❌❌✅✅✅❌

권한 적용 순서

Executor가 노드 실행 시도 │ ▼ PermissionChecker.require(capability) │ ├─ 권한 있음 → 런타임/툴/MCP 호출 진행 │ └─ 권한 없음 │ ├─ PermissionDeniedError 발생 ├─ PermissionDeniedEvent → EventBus → TraceWriter ├─ 해당 노드 NodeStatus.FAILED로 표시 └─ 나머지 노드는 계속 실행

커스텀 권한 세트 추가

# config/permissions.yaml
permission_sets:
  my_custom:
    file_read: true
    file_write: false
    shell_exec: false
    network_access: true
    mcp_call: true
    api_call: true
    memory_read: true
    memory_write: false
    sub_agent_delegate: false
# 에이전트 YAML에서 이름으로 참조
permissions: my_custom

인라인 권한 설정

세트 이름 대신 에이전트 YAML에 직접 지정할 수도 있습니다.

# agents/sub_agents/my_agent.yaml
permissions:
  file_read: true
  file_write: true
  shell_exec: false
  network_access: false
  mcp_call: false
  api_call: false
  memory_read: true
  memory_write: true
  sub_agent_delegate: false

Python에서 권한 확인

from core.permissions.base import Capability, PermissionSet
from core.permissions.checker import PermissionChecker, PermissionDeniedError

checker = PermissionChecker("my_agent", PermissionSet.read_only())

# 검사 후 예외 발생 (실행 막음)
checker.require(Capability.FILE_WRITE)  # → PermissionDeniedError

# 부울 반환 (조건부 처리)
if checker.check(Capability.NETWORK_ACCESS):
    # 네트워크 작업 수행
    ...
shell_exec는 신중하게 부여하세요 셸 실행 권한이 있는 에이전트가 악의적인 프롬프트 인젝션에 노출되면 시스템이 위험해질 수 있습니다. 반드시 필요한 에이전트에만, 샌드박스 모드로 부여하세요.
