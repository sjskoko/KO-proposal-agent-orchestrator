API 서버

FastAPI 기반 REST 인터페이스로 에이전트를 외부에서 호출하는 방법

서버 시작

# 개발 모드 (자동 리로드)
uvicorn apps.api_server.main:app --host 0.0.0.0 --port 8000 --reload

# 프로덕션
uvicorn apps.api_server.main:app --host 0.0.0.0 --port 8000 --workers 2

# Python 모듈로 직접
python -m apps.api_server.main

엔드포인트

메서드경로설명
POST/tasks작업 제출 — 동기 응답
GET/health서버 상태 확인
GET/docsSwagger UI (자동 생성)
GET/redocReDoc 문서 (자동 생성)

POST /tasks

요청 바디

{
  "goal": "수행할 작업 설명",     // 필수
  "agent_id": "main_agent"      // 선택 (기본값: main_agent)
}

응답

{
  "result": "에이전트 실행 결과 문자열",
  "session_id": "a1b2c3d4"      // 트레이스 조회에 사용
}

예시

# curl
curl -X POST http://localhost:8000/tasks \
  -H "Content-Type: application/json" \
  -d '{"goal": "Python 버블정렬 작성해줘", "agent_id": "coder"}'

# 응답
{
  "result": "def bubble_sort(arr):\n    n = len(arr)...",
  "session_id": "a1b2c3d4"
}

# Python httpx
import httpx
resp = httpx.post("http://localhost:8000/tasks",
    json={"goal": "파일 목록 알려줘", "agent_id": "file_worker"})
print(resp.json()["result"])

GET /health

curl http://localhost:8000/health
# 응답: {"status": "ok"}

서버 실행 중 브라우저에서 직접 테스트할 수 있습니다:

http://localhost:8000/docs

오류 응답

HTTP 코드상황

500에이전트 실행 중 예외 발생
# 422 응답 예시
{
  "detail": [

      "loc": ["body", "goal"],

      "type": "value_error.missing"
    }
  ]
