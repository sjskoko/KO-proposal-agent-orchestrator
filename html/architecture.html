레이어별 책임

앱 레이어 (apps/)

사용자 또는 외부 시스템의 진입점입니다. core의 어떤 것도 알지 못하고, 단순히 목표 문자열을 받아 Agent.run(goal)을 호출하고 결과를 출력합니다.

컴포넌트파일역할
CLIapps/cli/main.pyTyper 커맨드 라인 인터페이스
Runnerapps/cli/runner.py시스템 부트스트랩 (공유 로직)
REPLapps/cli/repl.py인터랙티브 루프
API 서버apps/api_server/main.pyFastAPI REST 엔드포인트

에이전트 코어 (core/agent/)

에이전트의 정체성과 생명주기를 담당합니다. YAML에서 정의를 로드하고, 상태머신으로 플래닝→실행→완료 흐름을 관리합니다.

파일역할
definition.pyYAML → AgentDefinition 파싱
state.pyidle/planning/executing/done 상태머신
delegation.py서브에이전트 위임 Protocol
base.pyAgent 클래스 — 모든 것을 묶는 오케스트레이터

플래너 (core/planner/)

목표 문자열을 받아 실행 가능한 단계 그래프(DAG)로 변환합니다. 모델에게 JSON 배열로 단계를 반환하도록 요청하고, 이를 TaskGraph로 변환합니다.

실행기 (core/executor/)

TaskGraph를 순회하며 준비된 노드를 실행합니다. 각 노드는 런타임 호출, 툴 호출, 서브에이전트 위임 중 하나입니다. 실패 시 지수 백오프로 재시도합니다.

런타임 레지스트리 (core/runtime/)

runtimes.yaml을 읽어 런타임 인스턴스를 lazy-load합니다. 실행기는 런타임 ID만 알고, 구현체는 모릅니다.

설계 원칙

원칙구현 방식
설정 주도core/ 안에 모델명, 툴명, 경로 하드코딩 없음
Protocol 경계모든 크로스-모듈 계약은 typing.Protocol
권한 강제런타임/MCP 호출 전 반드시 PermissionChecker.require()
런타임 격리한 런타임 크래시가 다른 런타임에 영향 없음
완전한 관측성모든 스텝이 이벤트 버스 → JSONL 파일로 기록
로컬 우선Gemma 4가 항상 fallback_chain 1순위
